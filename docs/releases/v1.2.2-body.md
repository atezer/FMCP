## v1.2.2 — 2026-04-01

### Plugin (F-MCP Bridge)

- **Multi-client `fileKey`:** The UI could open its WebSocket and send the first `ready` **before** the plugin main thread posted `FILE_IDENTITY`, so the bridge sometimes registered a client with **`fileKey: null`**. **`pushBridgeFileIdentity()`** re-sends `ready` after identity is known so `figma_list_connected_files` and per-`fileKey` routing work for every window (FigJam + multiple Figma browser tabs on one port).
