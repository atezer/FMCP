# v1.3.0 — Runtime port degisimi, coklu AI araci destegi

## Onemli degisiklikler

### `figma_set_port` araci (YENi)

Runtime'da WebSocket bridge portunu degistirebilirsiniz. Port mesgulse AI araci (Claude/Cursor) `figma_set_port(5456)` cagirarak baska bir porta gecer; Figma plugin'de ayni portu secince baglanti kurulur. Aralik: 5454-5470.

### Port catismasi dayanikliligi

Port mesgulse `process.exit(1)` yerine MCP stdio sunucusu ayakta kalir. `figma_get_status` uzerinden hata mesaji doner, `figma_set_port` ile farkli porta gecilebilir.

### Coklu AI araci (Claude + Cursor ayni anda)

Claude Desktop ve Cursor ayni makinede ayni anda calisabilir:
1. Ilk acilan varsayilan portu (5454) alir
2. Ikinci acildiginda `figma_get_status` "port mesgul" der
3. `figma_set_port(5456)` ile farkli porta gecilir
4. Figma plugin'de o port secilir — baglanti kurulur

### Diger

- `figma_get_status` genisledi: `bridgeListening`, `startError` alanlari
- `PluginBridgeServer` yeni API: `restart()`, `getPort()`, `isListening()`, `getStartError()`
- Onceki unreleased: sabit port stratejisi, graceful shutdown, probePort edge case
